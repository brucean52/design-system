# MLCheckbox

[Ant Documentation](https://3x.ant.design/components/checkbox)

## Usage Guidelines
Guidelines taken from [uxmovement.com](https://uxmovement.com/buttons/when-to-use-a-switch-or-checkbox/)

When to use a checkbox
* when the effect happens after pushing a 'save', 'submit' or other button for user to continue on journey (e.g. opting in for receiving emails when signing up for a new account)

When to use a [switch](http://marklogic-ui-component-library.eng.marklogic.com/?path=/story/data-entry-mlswitch--basic) instead of a checkbox
* to control something that will have visible instantaneous change (like a light switch, e.g. switching from light to dark mode)

## Text Guidelines
For field labels:
* Do not capitalize all letters.
* Use [Title Case](https://elementary.io/docs/human-interface-guidelines#capitalization) for phrases. Do not use punctuation for phrases.
* Use parallel structure when listing radios or check box options. If all options have the same introductory text, move that text to the group label.
Examples:

**Bad**

Search:

- [ ] When searching an indexed database include company owner
- [ ] When searching an indexed database include company name
- [ ] When searching an indexed database include industry name

**Good**

When searching an indexed database, include:

- [ ] Company Name
- [ ] Company Owner
- [ ] Industry Name
